## What went well
- **Grounding before acting.** Probing `create-hakit` (wanted the token at scaffold time → AD-8 conflict), reading `@hakit`'s installed types instead of guessing, checking Tailwind's *emitted* CSS — each caught a real disaster pre-ship.
- **AD-* spine + proactive guards were high-leverage.** Most "is this allowed?" answered itself. AD-8 guard (added in 1.1) kept paying off and made CI token-less for free.
- **Reuse discipline held.** One mapping (AD-7), one `useEntityValue` (AD-6), DRY `OfflineIcon`/`OfflinePill`. Epic 2 inherits infrastructure.
- **TDD earned its keep once it existed** (1.3+). The case-insensitive `Clock`/`clock` collision was caught by a test, not luck.
- **Pairing cadence, gating, and the BMAD create→dev→review→done loop worked well** (Florian's assessment).

## What didn't — honest patterns
- **Verified "emitted," not "wins."** Twice (1.2 stale-colour cascade, 1.6 skeleton) I confirmed code/CSS was present but not that it produced the right *outcome*. Both needed a code-review to catch.
- **Self-review < independent review.** I wrote *and* reviewed 1.5/1.6; reviews still found real self-introduced bugs (permanent skeleton, ref-during-render, cascade loss). Validates "review with a different model."
- **Deferred findings piled up untracked** (1.3 #2 shell-remount, 1.3 #5 tests-not-typechecked) — lived only in story notes; now captured in `TECH_DEBT.md`.
- **Nothing seen working on real HA.** 1.1/1.5/1.6 accepted with live-proof pending (Florian away). Foundation never ran E2E on real data this session — the biggest gap, and it's a *verification* gap, not a code one.
- **Infra/CI/deployment was discovered reactively, not architected** (Florian's key insight). The spine had AD-9 (serve from HA) but not the deploy mechanism, the LAN-only-vs-cloud-CI constraint, or CI/CD. All figured out *after* Epic 1.

## Lessons
1. **Verify the resolved outcome** (cascade winner, live re-render, which value wins), not just that code/CSS was emitted.
2. **Independent review beats self-review** for correctness — use a fresh context / different model on non-trivial logic.
3. **Track deferred findings the moment they're deferred** (ledger), or they rot.
4. **Live/device proof is a real gate** — don't let "done" outrun "seen working."
5. **Infra / CI / deployment belong in the architecture phase**, alongside app architecture — not discovered post-hoc.
